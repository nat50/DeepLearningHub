📚 Class Balance Analysis

What:

Check if classes are evenly represented in the dataset.

Why:

Imbalanced classes cause models to favor majority classes. Balanced data simplifies evaluation — accuracy is meaningful.

How:

Sort classes by sample count. A flat balance curve means perfect balance. Compute imbalance ratio = max_count / min_count.

import pandas as pd
import plotly.graph_objects as go

counts = df["class_index"].value_counts().sort_values(ascending=False)
fig = go.Figure(go.Bar(
    x=list(range(1, len(counts) + 1)),
    y=counts.values,
    marker_color="#6B9BD1",
))
fig.update_layout(title="Class Balance Curve (Sorted by Count)",
    xaxis_title="Class Rank", yaxis_title="Sample Count")
fig.show()

📚 Data Splits

What:

Partition data into training (70%), validation (15%), and test (15%) sets using stratified splitting.

Why:

Train on one subset, tune hyperparameters on another, and evaluate final performance on a held-out test set. Stratification ensures class proportions are preserved.

How:

Use sklearn.model_selection.train_test_split with stratify= parameter.

from sklearn.model_selection import train_test_split

train_df, temp_df = train_test_split(
    df, test_size=0.3, random_state=42, stratify=df["label"])
val_df, test_df = train_test_split(
    temp_df, test_size=0.5, random_state=42, stratify=temp_df["label"])

print(f"Train: {len(train_df):,}  Val: {len(val_df):,}  Test: {len(test_df):,}")

💡 Key Insights & Recommendations

▼

✅ Strengths

  • Perfectly balanced classes (140K each) — no resampling needed
  • Large dataset (1.4M samples) — ample for deep learning
  • Stratified splits preserve class distribution
  • Rich vocabulary diversity across categories

⚠️ Challenges

  • Noisy user-generated text (typos, slang, abbreviations)
  • Overlapping topics (e.g., Politics vs Society & Culture)
  • Missing content: 45% of question_content is null
  • Large vocabulary (2.16M unique words) requires truncation

💡 Recommendations

  • Use 256 token max length — covers most samples well
  • LSTM with 50K vocab balances coverage vs. embedding size
  • BERT tokenizer handles OOV via subword tokenization
  • Weighted metrics (precision, recall, F1) preferred for evaluation
