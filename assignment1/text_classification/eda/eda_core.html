📚 Class Distribution Analysis

What:

Counts of samples per topic category (10 classes).

Why:

Verify class balance — balanced datasets simplify training and metric interpretation.

How:

Group by class_index and count. Visualize with a bar chart.

import pandas as pd
import plotly.express as px

df = pd.read_csv("train.csv", header=None,
                  names=["class_index", "question_title", "question_content", "best_answer"])

class_names = {
    1: 'Society & Culture', 2: 'Science & Mathematics', 3: 'Health',
    4: 'Education & Reference', 5: 'Computers & Internet', 6: 'Sports',
    7: 'Business & Finance', 8: 'Entertainment & Music',
    9: 'Family & Relationships', 10: 'Politics & Government',
}

counts = df['class_index'].value_counts().sort_index()
px.bar(
    x=[class_names[i] for i in counts.index],
    y=counts.values,
    title="Class Distribution",
    labels={"x": "Category", "y": "Sample Count"},
).show()

Topic Category Distribution

Distribution of samples across 10 Yahoo Answers topic categories

📊 Key Insight

  • Perfectly Balanced: Each class has exactly 140,000 samples — no class imbalance to handle
  • Large Scale: 1.4M total samples provides ample data for deep learning models

📐 Word Count & Text Length Analysis

📚 Word Count Analysis

What:

Distribution of text lengths before and after preprocessing.

Why:

Helps choose max sequence length for LSTM/Transformer and understand data characteristics.

How:

Count words per sample with str.split().str.len(), visualize with histograms.

import pandas as pd
import plotly.graph_objects as go

df = pd.read_csv("train.csv", header=None,
                  names=["class_index", "question_title", "question_content", "best_answer"])
df["full_text"] = df["question_title"].fillna("") + " " + \
                  df["question_content"].fillna("") + " " + \
                  df["best_answer"].fillna("")
df["word_count"] = df["full_text"].str.split().str.len()

fig = go.Figure()
fig.add_trace(go.Histogram(x=df["word_count"], nbinsx=50, name="Raw",
    marker_color="#89B8E8", opacity=0.7))
fig.update_layout(title="Word Count Distribution",
    xaxis_title="Word Count", yaxis_title="Frequency")
fig.show()

Before Cleaning

Average: ~92 words/sample

After Cleaning

Average: ~41 words/sample (55.3% removed)

📐 Key Insights

  • Text Length: Average ~92 words raw, ~41 after cleaning — significant reduction
  • Max Sequence Length: 256 tokens covers the vast majority of samples

📚 Stop Word Analysis

What:

Proportion of stop words (the, is, of, etc.) in the corpus.

Why:

Stop words carry little semantic meaning but consume model capacity. Removing them improves signal-to-noise ratio.

How:

Match each word against a stop word set, calculate percentages.

import plotly.graph_objects as go

total_words = 128_189_357
stop_words_count = 64_416_436
meaningful = total_words - stop_words_count

fig = go.Figure(data=[go.Pie(
    labels=["Stop Words", "Meaningful Words"],
    values=[stop_words_count, meaningful],
    marker_colors=["#E8A4C4", "#89B8E8"],
    hole=0.4,
)])
fig.update_layout(title="Stop Words vs Meaningful Words")
fig.show()

📚 Vocabulary Richness per Category

📚 Vocabulary Richness

What:

Unique words and Type-Token Ratio (TTR) per category.

Why:

Higher TTR = more diverse vocabulary. Categories with rich vocabulary may have more distinctive features for classification.

How:

Count unique tokens per category; TTR = unique / total.

import pandas as pd
import plotly.express as px

# After loading and cleaning df
vocab_stats = df.groupby("class_name").agg(
    unique_words=("cleaned_text", lambda x: len(set(" ".join(x).split()))),
    total_words=("cleaned_text", lambda x: sum(len(t.split()) for t in x)),
).reset_index()
vocab_stats["ttr"] = vocab_stats["unique_words"] / vocab_stats["total_words"]

px.bar(vocab_stats, x="class_name", y="unique_words",
       title="Unique Words per Category").show()
px.bar(vocab_stats, x="class_name", y="ttr",
       title="Type-Token Ratio per Category").show()
